Features

The core of what this DAW consists of

Sequence

A channel rack with a 16 step sequencer, live linked patterns, and a piano roll with labeled keys, snapping, velocity editing, and note slicing.

Arrange

A timeline with MIDI and audio clips, marquee selection, clip duplication, automation lanes, and audio warping through the Rubber Band time stretcher.

Mix

Per track gain, pan, mute, and solo, buses and sends, plugin delay compensation, and built in EQ, compressor, limiter, delay, and reverb.

Host

VST3 and CLAP plugins behind one picker, each one sandboxed in its own process so a crashing plugin never takes your session down with it.

Launch

A session view with a clip launch grid and quantized scene launching, alongside track freezing for heavy projects.

Export

Deterministic offline rendering to 24 bit WAV. The same project renders the same audio, every time, on every platform.

How to Contribute

Howl is GPLv3 and built in the open. Bug reports, fixes, and features are all welcome.

Build it from source

You need CMake 3.22 or newer, Ninja, and a C++20 compiler. Every dependency is fetched and pinned automatically.

cmake -B build -G Ninja
cmake --build build

Learn the internals

The documentation page walks through how the code is laid out, how audio flows through the engine, and how that keeps it all working.

Report and discuss

Found a bug or want a feature? Open an issue. Changes land as pull requests against main and derivative work stays GPLv3.

FAQ

Short answers to the questions that come up first.

Is it really free?

Yes. Howl is free software under the GPLv3. There is nothing to buy, no locked features, and no subscription. You can use it, study it, change it, and share it.

What plugin formats does it support?

VST3 and CLAP, both behind one picker. VST2 is not supported and will not be. On Linux, Windows plugins can work through yabridge, with the usual caveats bridged plugins bring.

Which operating systems does it run on?

Linux, macOS, and Windows. All three are built and tested on every change.

Does it record audio?

No. Howl is sequencing first: MIDI, samples, plugins, and imported audio. Recording live input is not part of the current design. There is room for this to change in the future.

What can it export?

24 bit WAV through a deterministic offline renderer. Projects themselves save as plain JSON text, so they diff and version cleanly.

Where do I report bugs?

On the GitHub issue tracker. A short description and the steps to reproduce are all it takes.
